# 구글 인앱 결제 및 애드몹 연동 타임라인 (작업 순서)

이 문서는 프로젝트 시작부터 구글 플레이 스토어 최종 출시까지의 과정을 시간 순서대로 정리한 가이드입니다.

---

## 1단계: 초기 플랫폼 설정 (준비 단계)
개발 시작 전, 각 콘솔에서 필요한 ID를 확보합니다.
- [ ] **애드몹:** 앱 등록 및 '애드몹 앱 ID' 확보
- [ ] **애드몹:** 광고 단위 생성 (배너, 전면 등) 및 '광고 단위 ID' 확보
- [ ] **플레이 콘솔:** 앱 생성 및 기본 정보 설정

## 2단계: 프로젝트 환경 설정 (Android Studio)
코드 작성을 위한 기반을 마련합니다.
- [ ] `app/build.gradle`에 의존성 추가
    - `com.android.billingclient:billing-ktx` (결제)
    - `com.google.android.gms:play-services-ads` (광고)
- [ ] `AndroidManifest.xml` 설정
    - `APPLICATION_ID` 메타데이터에 애드몹 앱 ID 입력
    - `INTERNET`, `AD_ID` 권한 확인
- [ ] 개발용 테스트 광고 단위 ID 준비 (실제 ID 대신 사용)

## 3단계: 핵심 기능 및 UI 구현
광고 표시와 결제 로직을 코드로 구현합니다.
- [ ] **광고 초기화:** 앱 시작 시 `MobileAds.initialize` 호출
- [ ] **광고 UI:** 광고를 표시할 위치에 뷰 배치 및 로드 로직 작성
- [ ] **결제 로직:** `BillingClient` 초기화, 상품 조회, 결제 시작, 결과 처리 기능 구현
- [ ] **승인 및 복원:** `acknowledgePurchase`(승인) 및 `queryPurchasesAsync`(복원) 로직 구현
- [ ] **데이터 연동:** `isAdFree` 상태에 따라 광고 뷰를 숨기거나 보여주는 로직 연결

## 4단계: 인앱 상품 활성화 (최초 빌드 업로드)
결제 기능을 활성화하기 위해 구글 서버에 앱을 인식시킵니다.
- [ ] 결제 권한이 포함된 최초 빌드(AAB) 생성
- [ ] **플레이 콘솔:** 내부 테스트 트랙에 AAB 업로드
- [ ] **플레이 콘솔:** 업로드 후 '인앱 상품' 메뉴에서 `remove_ads` 상품 생성 및 **활성화** (빌드 업로드 전에는 생성 불가)

## 5단계: 통합 테스트 및 검증
실제 동작 여부를 확인합니다.
- [ ] **광고 테스트:** 테스트 ID로 광고가 정상적으로 나타나는지 확인
- [ ] **결제 테스터:** 플레이 콘솔에 '라이선스 테스터' 등록
- [ ] **결제 테스트:** 내부 테스트 트랙 앱에서 테스트 카드로 결제 진행
- [ ] **검증:** 결제 즉시 광고가 사라지는지, 앱 재설치 시 구매 복원이 되는지 확인

## 6단계: 최종 검토 및 출시
- [ ] 실제 애드몹 광고 단위 ID로 교체
- [ ] 프로덕션 트랙으로 앱 출시 승인 요청

---
> [!IMPORTANT]
> **주의:** 결제 승인(Acknowledge) 처리가 누락되면 구매 3일 후 자동 환불됩니다. 테스트 시 승인 상태가 `ACKNOWLEDGED`가 되는지 반드시 확인하세요.
